# 🚀 PY4E: Environment Setup & Progress Summary

This repository tracks the initial configuration and fundamental skills acquired during the setup phase of the **Python for Everybody (PY4E)** course. We have established two distinct workflows for writing and executing Python code.

---

## 🛠️ Development Environments

### 1. Cloud-Based: Replit
* **Access:** [replit.com/@ChuckSeverance/PY4E](https://replit.com/@ChuckSeverance/PY4E)
* **Setup:** Zero installation; runs in any web browser.
* **OS:** Uses a Linux-based container.
* **Best For:** Tablets, Chromebooks, or restricted work/school computers.

### 2. Local: Windows 11
* **Installation:** Python 3.x from [python.org](https://www.python.org).
* **Configuration:** Added Python to the system **PATH** for global command-line access.
* **Best For:** Professional offline development and high-performance tasks.

---

## 💻 Command Line Mastery

We have moved beyond the "Run" button to using the terminal. This allows for more complex program execution and server management.

| Action | Linux (Replit) | Windows (CMD) |
| :--- | :--- | :--- |
| **Check Current Folder** | `pwd` | `cd` |
| **List All Files** | `ls -l` | `dir` |
| **Change Directory** | `cd folder_name` | `cd folder_name` |
| **Go Up One Level** | `cd ..` | `cd ..` |
| **Run Python Script** | `python3 script.py` | `py script.py` |

> **Pro Tip:** Use the **Tab** key to auto-complete file names and the **Up-Arrow** to quickly re-run previous commands.

---

## 📝 Text Editors (The Programmer's Workbench)

We learned that coding requires **Plain Text**. Standard word processors (MS Word, WordPad) insert hidden formatting that breaks code.

* **Beginner:** **Thonny** (Simple, built-in debugger).
* **Intermediate:** **Nano** (Command-line based) or **Phoenix Code**.
* **Professional:** **VS Code** (Feature-rich with AI assistance).
* **Hard Mode:** **Vim** (Powerful, mode-based editing within the terminal).

---

## 🔄 The Developer's Cycle

Every program follows this repeatable workflow:

1.  **Write:** Author code in a plain text editor.
2.  **Save:** Store the file with a `.py` extension.
3.  **Navigate:** Use the command line (`cd`) to go to the file's location.
4.  **Run:** Execute the file using the Python interpreter.
5.  **Debug:** Fix any errors and repeat the cycle.
